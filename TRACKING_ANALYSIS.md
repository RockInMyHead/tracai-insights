# Анализ алгоритма трекинга TrackAI (видео от первого лица)

## Текущая архитектура

### 1. **Основные компоненты**

#### **A. Visual Odometry (VO) на основе признаков**
- **Детектор**: ORB (по умолчанию) или AKAZE (более точный, но медленнее)
- **Матчинг**: KNN + Lowe's ratio test (0.60) — фильтрует плохие совпадения
- **Геометрия**: MAGSAC+ / RANSAC для оценки гомографии/аффинного преобразования
- **Пороги качества**:
  - `min_good_matches = 70` (совпадения после ratio test)
  - `min_inliers_abs = 30` (минимум inliers от RANSAC)
  - `min_inlier_ratio = 0.45` (45% от src точек)

#### **B. Sparse Optical Flow (KLT tracking)**
- **Детекция**: `goodFeaturesToTrack` по 4×4 сетке (45 точек за ячейку)
- **Трекинг**: Lucas-Kanade с окном 21×21, 3 уровня пирамиды
- **Вывод**: вычисляется медиана движения в центре кадра
- **Вес**: 65% при успехе (если `sparse_ok=True`)

#### **C. Dense Optical Flow (Farneback)**
- **Использование**: резервный вариант (если feature VO не работает)
- **Параметры**: 2 уровня пирамиды, окно 9×9
- **Вес**: 20-80% в зависимости от confidence feature VO

#### **D. Фильтр Калмана**
- **Состояние**: [x, y, vx, vy]
- **Шумы**: процесс 0.01, измерение 0.5
- **Применение**: опционально (может добавлять lag)

#### **E. ROI (Область интереса)**
- **ML ROI** (YOLOv8s): детекция человека, bbox каждые 3 кадра
- **HOG fallback**: если ML ROI не включен
- **Фон**: MOG2 背景вычитание для маскирования статичных областей

#### **F. Маски сцены**
- **Egocentric**: обрезает края (8% верх, 12% низ, 4% стороны), обрезает область person ROI, фильтрует по текстуре
- **Person-centric**: использует bbox человека + foreground mask

---

## ✅ Что уже работает хорошо

1. **Прыгающие линии исчезли**
   - Двухуровневое сглаживание (буферы + экспоненциальное + Калман) стабилизирует позицию
   - Dead-zone (0.05 px/frame) предотвращает шум при низких скоростях

2. **Гибридный подход**
   - Комбинация sparse (KLT) + feature (VO) + dense (OF) обеспечивает resilience
   - Auto-switching между egocentric и person-centric режимами

3. **Robustness к плохим условиям**
   - Fallback на полный кадр если маска слишком бедна (< 80 признаков)
   - MOG2 + текстурный фильтр удаляют шумы

## ❌ Что может быть улучшено для повышения точности (видео от первого лица)

### **Критические проблемы**

#### **1. Очень строгие пороги RANSAC** → упущенные frames
```python
min_inliers_abs = 30       # Может быть слишком жесткий
min_inlier_ratio = 0.45    # 45% — много для первого лица
min_good_matches = 70      # 70 совпадений — требует много текстуры
```

**Рекомендация**:
```python
# При видео от первого лица часто мало четких совпадений (монотонный фон)
min_good_matches = 40-50           # Снизить до 40-50
min_inliers_abs = 20               # Снизить до 20-25
min_inlier_ratio = 0.30            # Снизить до 30-35% (более лояльно)
```

---

#### **2. Feature detector (ORB) чувствителен к масштабу и вращению**

**Проблема**: видео от первого лица имеет быстрые обзоры (rotation) и zoom-like эффект (близкие объекты).

**Рекомендация**:
- Использовать **AKAZE** по умолчанию вместо ORB (более инвариантен к вращению)
- Или добавить **ORB с большим `scaleFactor`** (текущий 1.15 → увеличить до 1.25)

```python
self.feature_detector = cv2.AKAZE_create(
    threshold=0.0001,      # Более чувствителен
    descriptor_size=0,
    descriptor_channels=3
)
```

---

#### **3. Маска сцены слишком агрессивна**

**Проблема**: обрезка 8% сверху + 12% снизу + 4% стороны может исключить важные features при быстром движении головы.

**Рекомендация**:
```python
# Reduce edge cuts при первом лице (больше полезной информации по краям)
top_cut = int(h * 0.04)     # Было 0.08 → 0.04
bottom_cut = int(h * 0.06)  # Было 0.12 → 0.06
side_cut = int(w * 0.02)    # Было 0.04 → 0.02
```

---

#### **4. Ratio test слишком строгий (0.60)**

**Проблема**: 0.60 отфильтровывает даже хорошие совпадения при бедной текстуре.

**Рекомендация**:
```python
# Lowe's ratio test можно ослабить для первого лица
good = [m[0] for m in matches if len(m) == 2 and m[0].distance < 0.70 * m[1].distance]
# Было 0.60 → 0.70 (немного лояльнее)
```

---

#### **5. Текстурный фильтр может быть слишком жесткий**

```python
self.scene_texture_threshold = 12  # Может исключать слегка текстурированные области
```

**Рекомендация**:
```python
self.scene_texture_threshold = 8-10  # Более лояльно к бедной текстуре
```

### **Вторичные улучшения**

#### **6. Sparse KLT трекинг недоис​пользуется**
- Текущий вес: 65% (если `sparse_ok`)
- Для первого лица (медленная, плавная сцена) KLT часто более надежен

**Рекомендация**:
```python
# При режиме egocentric и хорошем sparse трекинге повысить вес
if sparse_ok and mode == "ego":
    sparse_weight = 0.75 + 0.25 * sparse_conf  # Было 0.65 → 0.75
```

#### **7. Optical Flow (Farneback) параметры**

```python
flow_params = dict(
    pyr_scale=0.5,      # Может быть слишком грубым
    levels=2,           # Мало уровней
    winsize=9,          # Может быть больше для плавной сцены
)
```

**Рекомендация**:
```python
flow_params = dict(
    pyr_scale=0.5,
    levels=3,          # Увеличить до 3 (лучше многомасштабность)
    winsize=15,        # Увеличить до 15 (для плавной сцены лучше)
    iterations=5,      # Было 3 → 5
    poly_sigma=1.5,
)
```

---

#### **8. Слишком сильное сглаживание**

```python
alpha = 0.35  # Сглаживание при обновлении позиции
traj_smooth_alpha = 0.25  # Дополнительное сглаживание на выводе
```

**Проблема**: двойное сглаживание может вводить lag для резких движений.

**Рекомендация**:
```python
# Использовать ЛИШ ОДНО сглаживание, но более умно:
alpha = 0.40  # Экспоненциальное (было 0.35 → 0.40, менее агрессивное)
# traj_smooth_alpha оставить, но уменьшить:
traj_smooth_alpha = 0.15  # Было 0.25 → 0.15 (или убрать вообще)
```

---

#### **9. Straight-line lock слишком сильный**

```python
lateral_damping = 0.15  # Подавляет боковой дрейф на 15%
straight_mode_consistency = 0.96  # Очень высокий порог
```

**Рекомендация**:
```python
# Для первого лица это может "переис​правлять" повороты
lateral_damping = 0.08  # Было 0.15 → 0.08 (мягче)
straight_mode_consistency = 0.92  # Было 0.96 → 0.92 (легче активируется)
```

## 📋 План действий (приоритет)

### **High Priority (критичны для точности)**
1. ✅ Снизить пороги RANSAC (min_good_matches, min_inliers_abs, min_inlier_ratio)
2. ✅ Переключиться на AKAZE или увеличить scaleFactor ORB
3. ✅ Ослабить ratio test (0.60 → 0.70)
4. ✅ Уменьшить обрезку маски сцены (особенно низ/верх)

### **Medium Priority (качество + стабильность)**
5. Увеличить уровни Optical Flow (2 → 3)
6. Увеличить winsize OF (9 → 15)
7. Снизить aggressive сглаживание (alpha, traj_smooth_alpha)
8. Ослабить straight-line lock (lateral_damping, consistency)

### **Low Priority (fine-tuning)**
9. Повысить вес sparse KLT при успехе
10. Снизить texture threshold
11. Настроить параметры ML ROI (detect_interval, confidence)

## 🧪 Рекомендуемые тесты

1. **Лабиринт / коридор**: проверить, не уходит ли в стороны
2. **Резкие повороты**: KLT vs Feature VO
3. **Бедная текстура** (белые стены, монотонный пол): проверить fallback
4. **Быстрое движение**: оценить motion blur vs детекция

---

## Итог

Текущий алгоритм **хорошо справляется со стабильностью** (исчезли прыгающие линии), но **слишком консервативен в критериях качества**. Для видео от первого лица, где движение плавное, но текстура может быть бедной, нужно:

1. **Ослабить пороги** (особенно RANSAC)
2. **Повысить надежность KLT** (sparse tracking)
3. **Использовать более инвариантные детекторы** (AKAZE)
4. **Уменьшить aggressive сглаживание** (может вносить lag)

Эти изменения должны повысить точность **без потери стабильности**.
