## 4. Риски и рекомендации

### Риск 1: Таймаут загрузки 30 мин
- **Проблема:** При скорости 5–7 Мбит/с загрузка 2 ГБ > 30 мин.
- **Рекомендация:** Увеличить `xhr.timeout` до 60 мин (3600000 мс).

### Риск 2: Таймаут AVI-конвертации 30 мин
- **Проблема:** 2 ГБ AVI может конвертироваться 40–60 мин.
- **Рекомендация:** Увеличить `FFMPEG_TIMEOUT` до 3600 (1 ч) для AVI.

### Риск 3: Нет swap
- **Проблема:** При пиковой нагрузке возможен OOM.
- **Рекомендация:** Добавить swap 4–8 ГБ (опционально).

### Риск 4: Очистка старых файлов
- **Текущее:** `videos/` — 4.3 ГБ, `outputs/` — 5 МБ, `/tmp/trackai_temp` — 897 МБ.
- **Рекомендация:** Регулярно чистить старые видео или ввести политику хранения.

| Этап | MP4 | AVI |

| Загрузка (10 Мбит/с) | ~27 мин | ~27 мин |
| AVI→MP4 | — | 30–60 мин |

| Стабилизация | 15–25 мин | 15–25 мин |

| **Итого MP4** | **~50–80 мин** | — |
| **Итого AVI** | — | **~90–140 мин** |

Фронтенд ждёт до 60 мин — для AVI 2 ГБ этого может не хватить.

## 6. Рекомендуемые изменения

1. **Фронтенд:** `xhr.timeout` → 60 мин (3600000).
2. **Фронтенд:** `maxAttempts` → 3600 (120 мин ожидания).
3. **Бэкенд:** `FFMPEG_TIMEOUT` → 3600 для этапа AVI-конвертации.
4. **Мониторинг:** Логировать время каждого этапа для анализа.
